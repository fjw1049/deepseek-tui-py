3 · TUI 模式 tui/app.py · DeepSeekTUI

Textual App。compose 搭界面 → on_mount 起后台 worker _start_engine → Engine.create 完成后把 engine.run() 丢成 task，同时 _listen_events 消费事件队列刷新 UI。

sequenceDiagram
  participant U as 用户
  participant App as DeepSeekTUI (Textual)
  participant W as run_worker(_start_engine)
  participant E as Engine
  participant H as EngineHandle

  U->>App: 启动
  App->>App: compose() 搭 Sidebar/Transcript/Composer/StatusBar
  App->>W: on_mount → run_worker(_start_engine, exclusive)
  Note over W: 不阻塞 UI，先渲染欢迎页
  W->>E: Engine.create(handle, client, config)
  E-->>W: engine 就绪
  W->>App: self._engine = engine
  App->>E: asyncio.create_task(engine.run())
  App->>App: _flush_pending_messages()
  App->>H: _listen_events 监听 event 队列

  U->>App: 输入回车
  App->>App: _submit_user_message(text)
  App->>H: handle.send_message(text)  [入 op 队列]
  H->>E: next_op() 取到 SendMessageOp
  E->>E: _handle_send_message → TurnLoop
  E-->>H: emit TextDelta / ToolCall / ToolResult / TurnComplete
  H-->>App: _listen_events 收到 event
  App->>App: 刷新 Transcript / Sidebar / StatusBar

7 · Engine 主循环 Engine.run (orchestrator.py:897)
