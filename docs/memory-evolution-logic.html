Memory 与自主进化实现逻辑审核

审核对象为 2026-06-05 当前工作区内的 Smart Memory、Post-Turn 编排层与 Experience Evolution。 本文档按真实源码调用链整理，不只复述设计稿。

Memory 管“知道什么” Evolution 管“下次怎么做更好” PostTurn 管“什么时候学习” 默认关闭，配置开启 新增审核风险清单

1. 总结论

一句话： 这两套能力已经被接入同一个 turn 生命周期：每轮对话前召回经验，每轮对话后捕获/复盘经验，压缩或线程销毁前执行 flush 抢救。 Smart Memory 负责自动沉淀事实、偏好、场景和 persona；自主进化负责把“值得复用的做法”沉淀为 curated memory 或 procedural skill，并通过 ledger 审计。

入口

Engine.create() 创建 MemoryPipeline 与 EvolutionPipeline，再交给 PostTurnOrchestrator。

学习时机

普通轮次走 after_turn()；上下文压缩、退出、LRU 回收前走 flush_before_loss()。

写入策略

Memory 自动写 L0/L1/L2/L3；Evolution 先生成 mutation，再由 ledger 按策略 auto/propose/deny。

审核重点

当前完整三组测试有 1 个 L0 cursor 失败；另有几处行为边界需要明确。

2. 不要混淆的三个“记忆”

代码里同时存在三种 memory，作用不同，存储位置不同，进入 prompt 的方式也不同。

名称 用途 主要文件/目录 进入模型的方式
Legacy memory.md 人工维护的跨项目记忆，偏“用户可见笔记”。 ~/.deepseek/memory.md <user_memory> volatile block，manual/hybrid 模式注入。
Smart Memory 自动从对话中抽取结构化记忆、场景和 persona。 ~/.deepseek/memory_data/ 召回 L1 命中；追加 scene navigation 与 persona；必要时工具搜索。
Curated Memory 自主进化维护的精炼 agent 记忆和用户事实。 ~/.deepseek/memories/MEMORY.md、USER.md Session 启动时读取为稳定快照，作为系统 prompt 的 stable block。

3. 一轮对话的总链路

这是两套功能共同依附的主路径。先看这条链，再看各自内部实现。

1. Session 启动
engine/engine.py
创建 MemoryCoordinator；如果 evolution 开启，创建 EvolutionPipeline 并把 store/ledger 放进 ToolContext metadata。
2. 对话前召回
memory/coordinator.py
Smart Memory 根据用户输入搜索 L1；Curated Memory 的 stable snapshot 已缓存。
3. 组系统提示词
engine/prompts.py
顺序注入项目规则、环境、curated snapshot、evolution 指南、Smart Memory、skills、volatile 内容。
4. 工具循环
engine/turn_loop.py
模型可调用 memory_search / conversation_search / memory_curate / skill_manage。
5. TurnEvidence
post_turn/evidence.py
把 thread_id、user_text、workspace、新消息、工具轮数、成功状态打包为后处理输入。
6. Turn 后处理
post_turn/orchestrator.py
依次调用 MemoryPipeline 和 EvolutionPipeline；异常被记录，不打断主对话。

4. Smart Memory 实现逻辑

4.1 写入链路

步骤实现说明
Gate post_turn/gates.py 要求 turn 成功；有工具调用直接捕获；否则用户文本需超过最小长度且不是斜杠命令/简单确认。
L0 原始归档 memory/native/l0_recorder.py 把用户、assistant、tool 消息追加到 per-thread JSONL；tool 消息保留在 L0 但不送 L1 抽取。
L1 结构化抽取 memory/native/l1_extractor.py LLM 抽取 persona / episodic / instruction；质量 gate、置信度阈值、批量去重、安全扫描后写 SQLite。
L2 场景索引 memory/native/pipeline.py L1 写入后延迟聚合场景，生成可导航的 scene summary。
L3 Persona memory/native/l3_persona.py 按 interval 刷新长期用户画像，可结合场景导航。

4.2 召回链路

召回入口：MemoryCoordinator.recall_for_turn()，带 5 秒默认超时，失败只记录日志并降级为空。

结构化召回：NativeMemoryProvider.recall() 走 SQLite FTS，可选 hybrid embedding，根据 score threshold、limit、时间衰减返回 L1 命中。

Prompt 注入：L1 可以包进 user message，也可以放到 system volatile；scene navigation、persona、memory tools guide 追加到 system block。

主动下钻：memory_search 查 L1；conversation_search 查 L0 原始对话。每轮合计最多 3 次。

4.3 Smart Memory 的作用

跨会话记住事实

例如用户偏好、项目约定、曾经踩过的坑，通过 L1 结构化条目在后续对话召回。

保留原始证据

L0 JSONL 是审计和 conversation_search 的证据库，解决“结构化摘要不够时回查原文”的问题。

形成长期画像

L2/L3 把碎片化事件变成场景导航和 persona，让模型更稳定地理解用户和项目。

5. 自主进化实现逻辑

自主进化的核心抽象是 mutation。所有写入都先变成 ExperienceMutation，再交给 ledger 审批和审计。

1. 触发
evolution/pipeline.py
计数到期、工具轮数多、用户纠正、失败后恢复、显式要求记住流程，或 flush 前。
2. 后台 review
evolution/review/runner.py
复用 bounded tool loop，只开放 memory_curate / skill_manage，在 review_mode 下只产 mutation 不直接写盘。
3. Backend 转换
evolution/backends/*
memory_curate 变 curated memory mutation；skill_manage 变 skill mutation。
4. Ledger 决策
evolution/ledger.py
按策略写入 evolution_events：auto 自动 apply，propose 等用户批准，deny 拒绝。
5. 落盘
curated/store.py · skill_store.py
写 MEMORY.md / USER.md 或项目/用户 skills，并做审计与事件通知。
6. 后续生效
engine/prompts.py
Curated memory 下个 session 作为 stable snapshot 生效；skill 更新后提示模型 load_skill。

5.1 Curated Memory Backend

memory_curate 支持 add、replace、remove，目标为 memory 或 user。

  • MEMORY.md：agent 可复用的项目/工作方法事实。
  • USER.md：稳定用户偏好、身份、沟通习惯。
  • 条目用 § 分隔，写入前去重、限长、文件锁、原子替换。
  • Session 中 stable snapshot 冻结，mid-session 新写入不直接改变当前系统提示词。

5.2 Procedural Skill Backend

skill_manage 支持 create、patch、edit、delete、write_file、remove_file。

  • 默认写项目级 .deepseek/skills/<name>/SKILL.md。
  • 也可写用户级 ~/.deepseek/skills/。
  • 要求 SKILL.md 有 YAML frontmatter，且 name/description 合法。
  • 禁止绝对路径和 .. 越界，支持 references/templates/scripts/assets 辅助文件。

5.3 Ledger 状态机

策略返回记录状态后续行为默认配置
auto pending_apply → applied 或 failed 马上调用对应 backend 写盘。 memory_curate=auto，skill_patch=auto
propose proposed 发 EvolutionProposalEvent，等待用户 approve/reject。 skill_create=propose，多数非低风险 skill 变更走 propose。
deny denied 不写盘，只记录审计和 rejected event。 ledger disabled 时全部 deny。

6. 关键配置

配置默认值影响
[memory.smart].enabledfalse开启 Smart Memory 的 L0/L1/L2/L3 自动记忆。
[post_turn].enabledtrue开启统一 turn 后处理编排。关闭后 Evolution 不跑，Memory 有 Engine fallback 捕获。
[evolution].enabledfalse开启自主进化工具、后台 review、ledger 和 curated/skill stores。
memory_nudge_every_n10每 N 个 turn 触发一次 memory review 机会。
skill_nudge_tool_rounds10累计工具轮数达到阈值后触发 skill review 机会。
flush_min_user_turns6少于该 turn 数时，丢失前 evolution flush 不运行。
evolution.ledger.memory_curateautocurated memory 默认自动落盘。
evolution.ledger.skill_createpropose创建/删除/编辑 skill 默认需要提案审批。

7. 审核发现

以下是按源码和测试得出的风险点。优先级是对功能正确性/副作用影响的判断。

P1L0 cursor 在未来时间戳场景会重复记录旧 assistant 消息。

完整 memory/post_turn/evolution 测试中，test_l0_cursor_skips_already_recorded_messages 失败。 现实现用 append 调用时的 now_ms 写 cursor，但被记录消息可能自带更大的 timestamp； 下一轮判断 msg_ts <= last_ts 时旧消息没有被跳过，导致 L0 重复，进而可能让 L1 重复抽取。

相关文件：memory/native/l0_recorder.py、tests/memory/test_manifest_cleaner_seed.py。

P2主 agent 直接调用 memory_curate / skill_manage 时，用的是 turn 前半段 evidence。

Engine 在模型调用前先写入 TURN_EVIDENCE_KEY，此时 success=False 且消息只有用户输入； 之后工具调用可立即通过 ledger 自动落盘。若该 turn 后续失败或被取消，已经 auto applied 的 mutation 不会回滚。

相关文件：engine/engine.py、tools/memory_curate_tool.py、tools/skill_manage_tool.py、evolution/ledger.py。

P2review_idle_timeout_seconds 配置目前没有实际 idle 定时效果。

EvolutionPipeline 把该值传给 PeriodicTurnScheduler，但这个 scheduler 只实现计数阈值，没有 idle timer。 所以 evolution review 触发来自 turn 计数、工具轮数、signals 或 flush，不会因为空闲 600 秒自动运行。

相关文件：post_turn/scheduler.py、evolution/pipeline.py、config.example.toml。

P3工具返回给模型的 apply 细节偏少。

Ledger 会拿到 backend 的 ApplyResult，但 main tool response 构造时没有传入该结果。 memory_curate 还能返回 current_entries；skill_manage 在 applied 时多半只回 mutation target_path，可能不是实际 patch/write 的具体文件路径。

相关文件：evolution/tool_response.py、evolution/ledger.py、tools/skill_manage_tool.py。

P3安全扫描是基础防线，不是完整 prompt-injection 防线。

scan_memory_content() 当前只拦截少量正则模式，例如 <script、忽略之前指令、system: you are。 对 curated memory 和 skill 内容来说这是最低限度过滤，真正的安全仍依赖 ledger 策略、审批和人工审核。

相关文件：evolution/safety.py、curated/store.py、procedural/skill_store.py。
