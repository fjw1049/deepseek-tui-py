Sandbox 实现逻辑详解

deepseek-tui-py 当前实现：macOS Seatbelt（sandbox-exec）包裹 shell 子进程。 源码：execpolicy/sandbox.py + execpolicy/seatbelt.py， 接入点：tools/shell_tools.py。 生成时间：2026-05-27

L0 配置/合规 L1 应用层门控 L2 OS 沙箱 (Seatbelt) L3 升格 (设计，未实现) 不经 OS 沙箱

① 三层「沙箱」概念 必读

文档与代码里有多处 "sandbox"，含义不同，不可混用。

层级 名称 配置/入口 保护对象 Python 状态
L0 sandbox_mode config.toml / DEEPSEEK_SANDBOX_MODE 合规枚举、Workbench 展示、requirements 白名单 存储/校验，不驱动运行时 OS wrap
L1 应用层边界 ToolContext.resolve_path()、execpolicy、Plan registry 裁剪 路径逃逸、工具门控、命令黑名单 已实现
L2 Execution sandbox ExecutionSandboxPolicy → SandboxManager.prepare() 仅 shell spawn 已实现 (macOS Seatbelt)
L3 沙箱升格 elevated_sandbox_policy + retry API OS 拒绝后用户批准放宽策略重试 设计阶段，UI/API 未接
关键：运行时 shell 策略由 AppMode 决定（sandbox_policy_for_mode）， 不是 config.sandbox_mode。Rust parity 同样如此。

③ ExecutionSandboxPolicy 决策表

read-only

Plan 模式

  • should_sandbox() → true
  • 写盘 → 无 writable roots
  • 网络 → false
  • 全盘读 → true（所有策略均允许读）
workspace-write

Agent 模式（默认）

  • should_sandbox() → true
  • 可写根 → workspace + /tmp + $TMPDIR + writable_roots
  • .deepseek/ → 只读子路径
  • 网络 → true（agent 映射显式打开）
danger-full-access

Yolo / Trust 模式

  • should_sandbox() → false
  • 写盘 → 全盘
  • 网络 → true
  • 不 wrap sandbox-exec
external-sandbox

外部容器（预留）

  • should_sandbox() → false
  • 信任外层 OpenSandbox / 容器
  • 避免 double sandbox

Mode → Policy 映射（sandbox_policy_for_mode）

modePolicy.kindwritable_rootsnetwork_accessOS wrap
planread-only—false是
agentworkspace-write(workspace,)true是
yolo / trustdanger-full-access—true否

Policy 优先级（_resolve_policy）

1. override 参数（显式传入）
2. context.elevated_sandbox_policy   ← L3 升格重试时用
3. context.execution_sandbox_policy   ← Engine sync 写入
4. sandbox_policy_for_mode("agent", cwd)   ← 兜底

⑤ Seatbelt SBPL 生成（seatbelt.py）

generate_policy(policy, cwd) 按顺序拼接 SBPL 片段：

Layer A — SEATBELT_BASE_POLICY（所有模式共有）
  • (deny default) — 默认拒绝一切
  • 进程：process-exec, process-fork, signal, process-info*
  • 设备：/dev/null, /dev/urandom, /dev/ptmx, /dev/ttys*
  • 终端：pseudo-tty（PTY shell 必需）
  • IPC：ipc-posix-shm*, mach-lookup
  • sysctl-read, user-preference-read
Layer B — 全盘读（has_full_disk_read_access → 恒 true） 
(allow file-read*)

当前所有策略均允许读任意路径；写才受限制。

Layer C — 写策略 generate_write_policy()
policy.kindSBPL 写规则
danger-full-access / external(allow file-write* (regex #"^/"))
read-only（空，无写规则）
workspace-write见下方 Writable Roots 算法
 
; workspace-write 示例（单根，含 .deepseek 只读例外）
(allow file-write*
  (require-all
    (subpath (param "WRITABLE_ROOT_0"))
    (require-not (subpath (param "WRITABLE_ROOT_0_RO_0")))
  ))
Layer D — 网络（policy.has_network_access()） 
(allow network-outbound)
(allow network-inbound)
(allow system-socket)
(allow network-bind)

Agent 模式 network_access=True → 包含此段；Plan read-only → 不包含。

Layer E — 系统目录读 + Darwin Cache
  • DARWIN_USER_CACHE_DIR — read+write（via param）
  • 只读：/usr/lib, /usr/share, /System/Library, /Library/Preferences, /private/var/db
Layer F — Cargo 缓存规则 (#558 parity)
  • 始终允许读 $CARGO_HOME（默认 ~/.cargo）
  • 非 read-only 时额外允许写： $CARGO_HOME/registry、$CARGO_HOME/git
  • 解决 sandbox 内 cargo build 缓存失败

generate_params() — -D 参数绑定

参数名来源
WRITABLE_ROOT_{i}policy.get_writable_roots(cwd)[i].root canonical path
WRITABLE_ROOT_{i}_RO_{j}对应 root 的 read_only_subpaths[j]
DARWIN_USER_CACHE_DIRconfstr(_CS_DARWIN_USER_CACHE_DIR) 或 fallback ~/Library/Caches
CARGO_HOME$CARGO_HOME 或 ~/.cargo
CARGO_HOME_REGISTRY / _GITCARGO_HOME 子目录

⑥ get_writable_roots(cwd) 算法

仅 workspace-write 模式有效其他 kind → 返回 []
收集 Path 列表 roots[] ① policy.writable_roots ② cwd.resolve() ③ /tmp（除非 exclude_slash_tmp）④ $TMPDIR（除非 exclude_tmpdir）
去重（canonical path set）
每个 root → WritableRoot 若 {root}/.deepseek 目录存在 → read_only_subpaths += [.deepseek]
_propagate_read_only_subpaths() 嵌套 root：祖先 root 继承子 root 的 .deepseek 只读路径
返回 list[WritableRoot]
WritableRoot.is_path_writable(path)：路径在 root 下且不在任何 read_only_subpath 下 → 可写。 Seatbelt 在 SBPL 层用 require-not (subpath RO_param) 实现同等语义。

字段含义

denial_message 启发式

⑪ 实例：Agent 模式下 touch 命令的完整命令行

用户在工作区执行 touch ./foo.txt，mode=agent，macOS Seatbelt 可用：

/usr/bin/sandbox-exec \
  -p "(version 1)(deny default)...(allow file-read*)...(allow file-write* (subpath (param \"WRITABLE_ROOT_0\")))...(allow network-outbound)..." \
  -DWRITABLE_ROOT_0=/Users/dev/myproject \
  -DWRITABLE_ROOT_0_RO_0=/Users/dev/myproject/.deepseek \
  -DWRITABLE_ROOT_1=/tmp \
  -DDARWIN_USER_CACHE_DIR=/Users/dev/Library/Caches \
  -DCARGO_HOME=/Users/dev/.cargo \
  -DCARGO_HOME_REGISTRY=/Users/dev/.cargo/registry \
  -DCARGO_HOME_GIT=/Users/dev/.cargo/git \
  -- sh -c "touch ./foo.txt"

成功 → metadata: sandboxed=true, sandbox_type=macos-seatbelt, sandbox_denied=false

若改为 touch /etc/hosts → Seatbelt 拒绝 → sandbox_denied=true + denial_message
