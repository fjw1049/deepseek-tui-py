真续跑模型

Workflow / SubAgent / Task 三类执行体如何「从断点接着干」。 本文是实现前的契约：状态机、落盘边界、字段 schema、分工与分期。 打开即可看图；实现时以此为准，避免再做成「同 ID 重开」。

01 · 分工

三类执行体各续什么

不要三者抢同一套 cursor。续跑对象由执行原语决定：

flowchart LR
  subgraph pick ["怎么选"]
    Q["长任务 / 多步目标"]
  end
  Q --> R{类型}
  R -->|"声明式多步+扇出"| WF["Workflow
续：step / item"]
  R -->|"对话式探索"| SA["SubAgent
续：transcript 轮次"]
  R -->|"后台队列 turn"| TK["Task
续：transcript 轮次"]
  TK -.->|"prompt 含 workflow-detach"| WF
层执行原语真续跑含义不是真续跑
Workflow IR step / fanout item skip 已完成 step 与 {step}:{item} 只恢复 plan、重跑全部 inspect
SubAgent 自研 turn loop hydrate 已落盘 messages，从下一轮 LLM 继续 resume_agent 同 prompt 从 step 0 重开
Task（普通） Engine run_single_turn hydrate transcript，接着跑同一 turn 重启后重入队整段 prompt
Task（detach） Task 只当工人 断点只认 Workflow store Task 再造第二套 step cursor
一句话：Workflow 续的是计划图；SubAgent / Task 续的是同一条 transcript 的下一轮。 Detach 任务不双轨。
02 · 定义

什么叫「真续跑」

  1. 身份稳定：同一 run_id / agent_id / task_id，不是新建一条冒充继续。
  2. 跳过已完成工作：已 checkpoint 的 step/item 或已写入 transcript 的 tool 轮次不再重做。
  3. 状态干净：进入 running 时清掉旧的 error / sticky 取消文案。
  4. 入口明确：UI / 工具只走 resume(id)，禁止自由文本「继续」碰运气。
不保证：外部副作用事务回滚。半截 tool 的文件/网络改动不会自动撤销。 产品话术是「从上次完整工具轮之后继续」，不是「回滚后重放」。
03 · 边界

可恢复边界：完整 tool-round

SubAgent 与普通 Task 共用同一检查点粒度（Workflow 仍用 step/item）。

flowchart TD
  A["LLM 请求"] --> B["assistant 消息"]
  B --> C{"有 tool_calls?"}
  C -->|否| D["本轮结束 · 可终态"]
  C -->|是| E["逐个执行 tool"]
  E --> F["全部 tool_result 写入 messages"]
  F --> G["CHECKPOINT
落盘 transcript + cursor"]
  G --> A
  E -.->|中途 cancel/kill| X["本轮作废
保留上一完整 checkpoint"]
  • 只有「assistant + 全部 tool_result」齐了才写可恢复 checkpoint。
  • 工具执行到一半被杀 → 该轮不入档；续跑从上一完整轮之后的 LLM 请求开始。
  • 绝不在 tool 内部续跑。
04 · 状态机

统一终态与 resume 入口

stateDiagram-v2
  [*] --> running: spawn / create / resume
  running --> completed: 正常结束
  running --> failed: 执行错误
  running --> cancelled: 用户取消
  running --> timed_out: 墙钟超时
  running --> interrupted: 进程崩溃后加载
  cancelled --> running: resume(id)
  timed_out --> running: resume(id)
  interrupted --> running: resume(id)
  failed --> running: resume(id) 可选
  completed --> [*]
规则说明
timeout ≠ cancel墙钟超时必须独立终态（或 failed + 明确 error），禁止 cancel.set() 冒充取消。
resume 清 errorstatus=running 时 error=null，不残留 cancelled/timed_out 文案。
resume 接受默认：cancelled | timed_out | interrupted；failed 可配置；completed 用 restart 另开。
API 命名resume_* = 真续跑；restart_* = 清 transcript 同 id 重来（可选，后做）。
05 · SubAgent

Transcript 续跑

sequenceDiagram
  participant API as resume_agent
  participant M as SubAgentManager
  participant Disk as subagents + transcript
  participant L as run_subagent_loop
  API->>M: resume(agent_id)
  M->>Disk: load status + messages
  M->>M: status=running, clear result/error
  M->>L: drive with hydrated messages
  loop 每一完整 tool-round
    L->>Disk: checkpoint messages + steps
  end
  L-->>M: completed / cancelled / failed
  M->>Disk: 终态落盘

Loop 行为变更（相对现状）

  • 首次 spawn：messages = fork_messages? + user(prompt)（与今相同）。
  • resume：messages = 已落盘 transcript；不再重新 append 原始 user prompt。
  • input_queue：resume 开始前 drain，作为新 user message 注入（顺带修空壳）。
  • 嵌套子 agent：父 resume 不自动 resume 子；子各自有 id 与终态。

落盘位置（建议）

06 · Task

普通 Task vs Detach 双轨

flowchart TD
  T["task 出队 running"] --> D{"prompt 是 workflow-detach?"}
  D -->|是| WF["execute_detached_workflow
skip_step_ids + outputs"]
  D -->|否| E["Engine turn
hydrate transcript?"]
  E --> R["每轮 tool-round 刷盘"]
  R --> F["终态: completed / failed / cancelled / timed_out"]
  WF --> F
  F -->|可 resume| RES["task_resume(id)"]
  RES --> D

普通 Task

  • 新增 task_resume(id)；进程恢复时：有 transcript → 续跑，无 → 兼容旧行为或标记不可续。
  • Checklist / timeline = UI 与模型旁路，不是执行 cursor；续跑首轮可把 checklist 摘要注入 nudge。
  • Engine 今日每次新建：resume 时 hydrate session messages 再继续，而不是再喂原始 prompt 开新 turn。

Detach Task

  • 断点只在 .deepseek/workflow-runs/<run_id>/run.json。
  • Task 层负责队列、cancel token、把 timeout/cancel 正确映射到 workflow 终态。
  • 禁止给 detach 任务再挂一份 Engine transcript 当第二真相。
现状债（落地前必清）：墙钟超时里 cancel.set()，finalize 走 canceled 并丢掉 Task timed out after …。不先拆开 timeout，续跑入口会脏。

08 · UI

可视化与操作契约

后续做面板时按此契约，避免再出现 Workflow 那种「聊天继续 ≠ 续跑」。

flowchart TB
  subgraph tray ["ProcessTray / Dock"]
    C["终态卡 cancelled / timed_out"]
    B["续跑按钮"]
  end
  B --> P{"kind"}
  P -->|workflow| W["sendMessage: workflow run_id only"]
  P -->|subagent| S["resume_agent agent_id"]
  P -->|task 普通| T["task_resume task_id"]
  P -->|task detach| D["workflow run_id 续跑
或 task_resume 走 detach 分支"]
  C --> ID["同一 id 只保留一张卡
running 优先折叠旧终态"]
  • 按钮文案区分：续跑 vs 重新开始（后者清 transcript / 新 run）。
  • 进度可视化：Workflow 显示 phase/item；SubAgent/Task 显示 steps + 最近 tool；checklist 仅 Task 旁路。
  • 双徽章规则：同 id 出现新 running 时折叠旧终态（Workflow 已做，另两类对齐）。
09 · 分期

落地顺序（实现清单）

  P0["Phase 0

  P1 --> P2["Phase 2
Task transcript + API"]
  P2 --> P3["Phase 3

Phase做什么验收
0 Task 超时≠取消；文档/命名区分 resume vs restart 超时任务 status/error 可见 timed out；不进 canceled
1 SubAgent 落盘/恢复 transcript；loop hydrate 取消后 resume 不重跑已完成 tool 轮；单测覆盖
2 普通 Task transcript + task_resume；detach 仍只走 Workflow 中断后台 turn 可续；detach 仍 skip step/item
3 UI 按钮只发 resume(id)；Dock/Tray 折叠规则对齐 点续跑 ≠ 自由文本继续；无双徽章
4 checklist nudge、input_queue、可选 restart API 体验项，不阻塞真续跑
明确不做（除非另开设计）：按 checklist item 当执行 cursor 跳工具； 在 tool 内部续跑；给 detach Task 再挂 Engine transcript。
