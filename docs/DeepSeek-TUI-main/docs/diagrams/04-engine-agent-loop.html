🔄 Engine 主循环流程

📋 Op 操作枚举

SendMessage

发送用户消息，启动新的 Turn 循环。包含 content 和 mode 参数。

CancelRequest

取消当前正在执行的请求，触发 cancel_token。

ApproveToolCall

批准指定 ID 的工具调用，继续执行。

DenyToolCall

拒绝指定 ID 的工具调用，返回错误。

SpawnSubAgent

生成子 Agent，包含 prompt 和 config 参数。

ListSubAgents

列出所有活跃的子 Agent。

ChangeMode

切换 AppMode (Agent/Yolo/Plan)。

SetModel

动态切换 LLM 模型。

SetCompaction

启用/禁用上下文压缩。

SyncSession

同步会话状态到持久化存储。

CompactContext

手动触发上下文压缩。

Rlm

RLM (Reasoning Language Model) 操作。

EditLastTurn

编辑最后一个 Turn 的内容。

Shutdown

关闭 Engine，清理资源。

🔧 系统提示构建

系统提示组成部分
1. 基础系统提示 (Mode-specific) prompts::system_prompt_for_mode_with_context_and_skills() - Agent Mode: 完整能力 + 审批流程 - Yolo Mode: 自动执行 + 无审批 - Plan Mode: 规划优先 + 分步执行 2. Working Set 摘要 - 当前工作目录文件列表 - 最近修改的文件 - Git 状态信息 3. 用户记忆块 (#489) - 用户偏好设置 - 历史交互记忆 - 上下文相关信息 4. 自定义指令 (#454) - 用户定义的行为规则 - 特定任务指令 - 约束条件 5. 压缩摘要 (如有) - 上下文恢复时合并 - 历史对话摘要 - 关键信息保留

🔄 Turn 循环编排

handle_deepseek_turn() 核心逻辑

透明流重试

MAX_TRANSPARENT_STREAM_RETRIES = 2 仅在未收到任何内容时重试 避免部分响应丢失

连续工具错误追踪

consecutive_tool_error_steps 计数器 超过阈值时中断循环 防止无限错误循环

上下文恢复

MAX_CONTEXT_RECOVERY_ATTEMPTS = 3 处理上下文溢出 自动压缩并重试

Steer 输入

从 rx_steer 排空消息 Turn 中途注入用户消息 作为 User 角色添加到历史

压缩检查点

超过阈值自动压缩 保持上下文窗口可用 保留关键信息

容量预请求检查

API 调用前的护栏决策 评估风险等级 决定是否继续

LSP 诊断刷新

合成用户消息包含编译错误 自动注入诊断信息 辅助 LLM 修复错误

分层上下文接缝检查点 (#159)

追加 <archived_context> 块 保留历史上下文结构 支持多层压缩

⚖️ 容量控制器集成

风险等级与检查点

风险等级

Low 低风险

正常操作，无需特殊处理。

Medium 中风险

需要监控，可能需要用户确认。

High 高风险

需要明确审批，记录详细日志。

Critical 严重

阻止执行，需要管理员权限。

检查点类型

1. 预请求检查点 (Pre-request Checkpoint) - API 调用前的护栏决策 - 评估当前上下文容量 - 决定是否继续或压缩 2. 干预检查点 (Intervention Checkpoint) - Turn 执行期间的动态检查 - 监控资源使用 - 必要时中断执行 3. 容量记忆 (Capacity Memory) - 恢复时的上下文状态 - 历史容量使用记录 - 优化未来决策
