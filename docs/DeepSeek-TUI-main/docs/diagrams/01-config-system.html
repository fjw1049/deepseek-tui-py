1. 加载优先级链 (Loading Precedence Chain)

配置值按以下优先级从高到低合并，高优先级覆盖低优先级：

1
CLI 参数
--model, --provider, --api-key, --base-url
2
环境变量
DEEPSEEK_*, NVIDIA_API_KEY, OPENAI_BASE_URL
3
config.toml
~/.deepseek/config.toml, ./config.toml
4
默认值
Config::default() 硬编码
📋 优先级示例
# 场景：用户同时设置了多个配置源 # CLI: --model deepseek-v4-flash # ENV: DEEPSEEK_MODEL=deepseek-v4-pro # TOML: model = "gpt-4.1" # 默认: deepseek-v4-pro # 最终生效：deepseek-v4-flash (CLI 优先级最高)

2. Profile 系统 (Profile Merging)

Profile 允许为不同环境（dev/prod/work）维护独立配置集。

1 Profile 选择
# 通过 CLI 参数 deepseek --profile work # 或环境变量 export DEEPSEEK_PROFILE=dev
2 TOML Profile 定义
[profiles.dev] api_key = "DEV_KEY" provider = "deepseek" allow_shell = true [profiles.prod] api_key = "PROD_KEY" provider = "nvidia-nim" approval_policy = "on-request"
3 深度合并策略
  • 标量字段：Profile 值覆盖基础配置
  • 数组字段：完全替换（不追加）
  • Provider 映射：字段级合并（可只覆盖 model 而保留 api_key）
🔍 合并逻辑详解
// crates/config/src/lib.rs:251-316 pub fn merge_project_overrides(&mut self, project: ConfigToml) { // 标量字段：有值则覆盖 if project.api_key.is_some() { self.api_key = project.api_key; } // Provider 子表：字段级合并 merge_provider_config(&mut self.providers.deepseek, &project.providers.deepseek); }

3. Provider 配置 (Provider Configuration)

支持多 Provider 并存，通过 provider 字段切换。

ProvidersToml
deepseek: ProviderConfigToml // DeepSeek Platform
nvidia_nim: ProviderConfigToml // NVIDIA NIM
openai: ProviderConfigToml // OpenAI
openrouter: ProviderConfigToml // OpenRouter
novita: ProviderConfigToml // Novita
ProviderConfigToml
api_key: Option<String>
base_url: Option<String>
model: Option<String>
📝 Provider 切换示例
# config.toml 中预存多个 Provider provider = "deepseek" [providers.deepseek] api_key = "sk-deepseek-xxx" model = "deepseek-v4-pro" [providers.nvidia_nim] api_key = "nvapi-xxx" base_url = "https://integrate.api.nvidia.com/v1" model = "deepseek-ai/deepseek-v4-pro" # 运行时切换（无需重新输入密钥） deepseek --provider nvidia-nim

4. 环境变量映射 (Environment Variable Mapping)

环境变量通过特定前缀和分隔符映射到配置字段。

环境变量 映射字段 类型 说明
DEEPSEEK_PROVIDER Config.provider ProviderKind 全局 Provider 选择
DEEPSEEK_MODEL Config.model String 默认模型名
DEEPSEEK_API_KEY 通过 Secrets 解析 String API 密钥（优先级低于 keyring）
DEEPSEEK_BASE_URL Config.base_url String DeepSeek API 端点
NVIDIA_API_KEY 通过 Secrets 解析 String NVIDIA NIM 密钥
NVIDIA_NIM_BASE_URL EnvRuntimeOverrides.nvidia_base_url String NIM 端点（别名：NIM_BASE_URL）
OPENAI_BASE_URL EnvRuntimeOverrides.openai_base_url String OpenAI 兼容端点
DEEPSEEK_TELEMETRY Config.telemetry bool 遥测开关（1/true/yes → true）
🔧 环境变量解析逻辑
// crates/config/src/lib.rs:902-948 impl EnvRuntimeOverrides { fn load() -> Self { Self { provider: std::env::var("DEEPSEEK_PROVIDER") .ok() .and_then(|v| ProviderKind::parse(&v)), model: std::env::var("DEEPSEEK_MODEL").ok(), telemetry: std::env::var("DEEPSEEK_TELEMETRY") .ok() .and_then(|v| parse_bool(&v).ok()), // ... } } }
