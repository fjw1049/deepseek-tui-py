1️⃣ HTTP 请求构建流程

步骤 1: 构建 MessageRequest

Engine 引擎根据用户输入和系统提示构建请求对象

MessageRequest { model: "deepseek-chat", messages: Vec<Message>, max_tokens: 8192, system: Option<String>, tools: Vec<Tool>, reasoning_effort: Option<String>, stream: true }

步骤 2: 转换为 ChatCompletionRequest

DeepSeekClient 将内部格式转换为 OpenAI 兼容格式

ChatCompletionRequest { model: String, messages: Vec<ChatMessage>, stream: true, tools: Option<Vec<ChatTool>>, temperature: Option<f32>, max_tokens: Option<u32> }

步骤 3: 工具名称编码

通过 to_api_tool_name() 将工具名称转换为 API 兼容格式

编码规则: - 字母数字 + '_' → 直接通过 - '-' → '--' (双短横线) - 其他字符 → '-x<hex>-' (十六进制编码) 示例: "read-file" → "read--file" "web@fetch" → "web-x40-fetch"

步骤 4: HTTP POST 请求

发送到 DeepSeek API 端点

POST https://api.deepseek.com/v1/chat/completions Headers: Authorization: Bearer <api_key> Content-Type: application/json Body: JSON 序列化的 ChatCompletionRequest

2️⃣ SSE 流式解析

Server-Sent Events (SSE) 格式

  • 响应格式: data: {json}\n\n
  • 每个事件以双换行符分隔
  • 流结束标记: data: [DONE]
  • 使用 eventsource_stream crate 解析
📋 SSE 事件类型
事件类型 触发时机 携带数据
MessageStarted 消息开始 message_id, role
MessageDelta 内容增量 delta (text/thinking/tool_call)
MessageComplete 消息结束 stop_reason, usage
ThinkingStarted 思考块开始 block_index
ThinkingDelta 思考内容增量 delta (thinking text)
ThinkingComplete 思考块结束 -
ToolCallStarted 工具调用开始 tool_call_id, tool_name
ToolCallComplete 工具调用结束 tool_call_id, input (JSON)
TurnStarted 回合开始 turn_index
TurnComplete 回合结束 -

Delta 清理: sanitize_stream_chunk()

移除控制字符,确保文本安全

fn sanitize_stream_chunk(text: &str) -> String { text.chars() .filter(|c| !c.is_control() || *c == '\n' || *c == '\t') .collect() }

工具 JSON 缓冲

累积 input_buffer 直到完整 JSON 可解析

ToolUseState { id: String, name: String, input_buffer: String, // 累积原始 JSON input: Option<Value>, // 解析后的 JSON }

4️⃣ 超时 & 重试逻辑

⏱️ 超时配置

  • STREAM_CHUNK_TIMEOUT_SECS = 90
  • 空闲超时,每收到 SSE 块重置
  • STREAM_MAX_DURATION_SECS = 1800
  • 墙钟时间上限 (30 分钟)
  • STREAM_MAX_CONTENT_BYTES = 10 MB
  • 内容大小安全上限

🔄 重试预算

  • MAX_STREAM_ERRORS_BEFORE_FAIL = 5
  • 连续可恢复错误次数
  • MAX_TRANSPARENT_STREAM_RETRIES = 2
  • 透明重试次数 (无内容时)
  • 仅在未收到内容时透明重试
🔍 错误分类
错误类型 可恢复 处理策略
网络超时 ✅ 是 重试 (计入预算)
连接中断 ✅ 是 透明重试 (如无内容)
解码错误 ✅ 是 跳过该块,继续
认证失败 ❌ 否 立即失败
上下文溢出 ❌ 否 立即失败
模型错误 ❌ 否 立即失败

透明重试条件: should_transparently_retry_stream()

fn should_transparently_retry_stream( no_content_received: bool, retry_count: usize, cancelled: bool ) -> bool { no_content_received && retry_count < MAX_TRANSPARENT_STREAM_RETRIES && !cancelled }

5️⃣ 伪造工具调用清理

⚠️ 问题背景

  • 某些 LLM 会在文本中伪造工具调用标记
  • 这些标记不是真实的工具调用,只是文本内容
  • 需要过滤以避免混淆用户
🏷️ 工具调用标记列表

开始标记

[TOOL_CALL] <deepseek:tool_call <tool_call <invoke <function_calls>

结束标记

[/TOOL_CALL] </deepseek:tool_call> </tool_call> </invoke> </function_calls>

filter_tool_call_delta() 函数

从文本增量中剥离伪造的工具调用包装器

fn filter_tool_call_delta(text: &str) -> (String, bool) { let mut filtered = text.to_string(); let mut scrubbed = false; for marker in TOOL_CALL_START_MARKERS { if filtered.contains(marker) { filtered = filtered.replace(marker, ""); scrubbed = true; } } for marker in TOOL_CALL_END_MARKERS { if filtered.contains(marker) { filtered = filtered.replace(marker, ""); scrubbed = true; } } (filtered, scrubbed) }

🔔 用户通知

  • 当检测到伪造标记时,显示 FAKE_WRAPPER_NOTICE
  • 通知用户某些内容已被过滤
  • 保持透明度,避免混淆

📦 关键数据结构

MessageRequest
struct MessageRequest { /// 模型名称 (如 "deepseek-chat") model: String, /// 消息历史 messages: Vec<Message>, /// 最大生成 token 数 max_tokens: u32, /// 系统提示 system: Option<String>, /// 可用工具列表 tools: Vec<Tool>, /// 推理努力程度 (DeepSeek 特有) reasoning_effort: Option<String>, /// 是否流式响应 stream: bool, }
ChatCompletionRequest (OpenAI 兼容)
struct ChatCompletionRequest { model: String, messages: Vec<ChatMessage>, stream: bool, tools: Option<Vec<ChatTool>>, temperature: Option<f32>, max_tokens: Option<u32>, top_p: Option<f32>, frequency_penalty: Option<f32>, presence_penalty: Option<f32>, }
ContentBlockKind
enum ContentBlockKind { /// 普通文本内容 Text, /// 推理思考内容 (DeepSeek 特有) /// 显示模型的思考过程 Thinking, /// 工具调用 /// 包含工具名称和参数 ToolUse, }
ToolUseState
struct ToolUseState { /// 工具调用 ID (由 LLM 生成) id: String, /// 工具名称 (已从 API 格式解码) name: String, /// 解析后的 JSON 输入参数 input: serde_json::Value, /// 调用者元数据 (可选) caller: Option<ToolCaller>, /// 原始 JSON 累积缓冲区 /// 用于增量接收 JSON 片段 input_buffer: String, }
StreamEvent (SSE 事件)
enum StreamEvent { MessageStarted { id: String, role: String }, MessageDelta { delta: ContentDelta }, MessageComplete { stop_reason: String, usage: Usage }, ThinkingStarted { index: usize }, ThinkingDelta { delta: String }, ThinkingComplete, ToolCallStarted { id: String, name: String }, ToolCallComplete { id: String, input: Value }, TurnStarted { index: usize }, TurnComplete, }
Usage (Token 统计)
struct Usage { /// 输入 token 数 (prompt) prompt_tokens: u32, /// 输出 token 数 (completion) completion_tokens: u32, /// 总 token 数 total_tokens: u32, /// 缓存命中 token 数 (可选) prompt_cache_hit_tokens: Option<u32>, /// 缓存未命中 token 数 (可选) prompt_cache_miss_tokens: Option<u32>, }
