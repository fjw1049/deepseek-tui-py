📋 CLI 参数解析

子命令 (Subcommands)

  • run - 启动交互式 TUI 或 Agent 模式
  • prompt - 单次提示执行 (非交互)
  • export - 导出会话历史
  • config - 配置管理命令
  • version - 显示版本信息

全局标志 (Global Flags)

  • --model <MODEL> - 指定 LLM 模型 (覆盖配置)
  • --provider <PROVIDER> - 指定 Provider (deepseek/openai/anthropic)
  • --profile <PROFILE> - 使用配置 Profile
  • --config <PATH> - 指定配置文件路径
  • --verbose, -v - 启用详细日志
  • --no-color - 禁用彩色输出

run 子命令选项

  • --mode <MODE> - 运行模式: agent (默认) / yolo / plan
  • --workspace <PATH> - 工作目录路径
  • --auto-approve - 自动批准所有工具执行
  • --server - 启动 HTTP 服务器模式而非 TUI

prompt 子命令选项

  • --input <TEXT> - 输入提示文本
  • --output <PATH> - 输出文件路径
  • --format <FORMAT> - 输出格式: json / plain

⚙️ 配置加载流程

Config::load() 优先级链
  1. 加载默认值 - 内置的 Default::default()
  2. 发现配置文件 - 按顺序查找:
    • --config 参数指定的路径
    • 当前目录 ./config.toml
    • $HOME/.deepseek/config.toml
    • XDG_CONFIG_HOME/deepseek/config.toml
  3. 解析 TOML - toml::from_str() 反序列化
  4. 合并 Profile - 如果指定 --profile, 合并对应配置块
  5. 应用环境变量 - DEEPSEEK_* 前缀覆盖
  6. 应用 CLI 覆盖 - 命令行参数优先级最高
  7. 验证配置 - 检查必需字段和合法性
配置文件结构示例
[default] model = "deepseek-chat" provider = "deepseek" temperature = 0.7 max_tokens = 4096 [providers.deepseek] base_url = "https://api.deepseek.com/v1" timeout = 90 [tools] enabled = ["bash", "read", "write", "edit"] timeout = 30 [profiles.fast] model = "deepseek-chat" temperature = 0.3 [profiles.creative] model = "deepseek-chat" temperature = 1.0

🔐 Secrets 初始化

Secrets::load() 优先级

  1. 环境变量 (最高优先级)
    • DEEPSEEK_API_KEY
    • OPENAI_API_KEY
    • ANTHROPIC_API_KEY
    • NVIDIA_NIM_API_KEY
    • OPENROUTER_API_KEY
  2. OS Keyring (推荐)
    • macOS: Keychain Access
    • Windows: Credential Manager
    • Linux: Secret Service (libsecret)
  3. config.toml (不安全的回退)
    • [secrets] 配置块
    • 明文存储, 仅用于开发环境
Keyring 存储示例 (macOS)
# 存储 API Key 到 Keychain security add-generic-password \ -a "$USER" \ -s "deepseek-tui-api-key" \ -w "sk-xxxxxxxxxxxxx" # 读取 API Key security find-generic-password \ -a "$USER" \ -s "deepseek-tui-api-key" \ -w

💾 State 初始化

State::open() 流程

  • 打开 SQLite 数据库: ~/.deepseek/state.db
  • 检查 schema 版本, 执行迁移 (如需要)
  • 创建表 (如不存在):
    • sessions - 会话元数据
    • messages - 消息历史
    • checkpoints - 状态快照
    • queued_requests - 离线队列
  • 崩溃恢复: 检测未正常关闭的会话, 从 checkpoint 恢复
数据库 Schema
CREATE TABLE sessions ( id TEXT PRIMARY KEY, created_at INTEGER NOT NULL, updated_at INTEGER NOT NULL, mode TEXT NOT NULL, metadata TEXT ); CREATE TABLE messages ( id INTEGER PRIMARY KEY AUTOINCREMENT, session_id TEXT NOT NULL, role TEXT NOT NULL, content TEXT NOT NULL, timestamp INTEGER NOT NULL, FOREIGN KEY (session_id) REFERENCES sessions(id) ); CREATE TABLE checkpoints ( id INTEGER PRIMARY KEY AUTOINCREMENT, session_id TEXT NOT NULL, state_json TEXT NOT NULL, timestamp INTEGER NOT NULL, FOREIGN KEY (session_id) REFERENCES sessions(id) );

AgentRegistry::new() 流程

🖥️ TUI 启动流程

run_tui() 详细步骤

  1. 启用原始模式
    crossterm::terminal::enable_raw_mode()?
    禁用行缓冲, 捕获每个按键
  2. 进入备用屏幕
    crossterm::execute!(stdout(), EnterAlternateScreen)?
    保留原终端内容, 使用独立缓冲区
  3. 创建 Terminal
    ratatui::Terminal::new(CrosstermBackend::new(stdout()))?
    初始化 ratatui 渲染管线
  4. 生成 Engine 后台任务
    tokio::spawn(spawn_engine(config, state))
    异步运行 Engine, 处理 Op 调度
  5. 运行事件循环
    run_event_loop(terminal, app_state).await?
    48ms 轮询键盘事件, 24ms 渲染更新
TUI 清理流程 (退出时)
// 1. 退出事件循环 event_loop.stop(); // 2. 离开备用屏幕 crossterm::execute!(stdout(), LeaveAlternateScreen)?; // 3. 禁用原始模式 crossterm::terminal::disable_raw_mode()?; // 4. 保存会话状态 state.save_checkpoint(session_id)?; // 5. 关闭数据库连接 state.close()?;

🌐 AppServer 启动流程

run_server() 详细步骤

  1. 创建 axum Router
    let app = Router::new()
    初始化 HTTP 路由器
  2. 注册路由
    • POST /v1/chat/completions - OpenAI 兼容 API
    • GET /v1/sse - Server-Sent Events 流
    • GET /health - 健康检查
    • GET /metrics - Prometheus 指标
  3. 配置中间件
    .layer(CorsLayer::permissive())
    .layer(TraceLayer::new_for_http())
    CORS 和请求追踪
  4. 绑定地址
    let addr = SocketAddr::from(([127, 0, 0, 1], 3000))
    默认监听 127.0.0.1:3000
  5. 启动服务器
    axum::Server::bind(&addr).serve(app.into_make_service()).await?
    阻塞运行, 处理 HTTP 请求

⚠️ 错误处理

缺失 API Key

  • 检测: Secrets::load() 返回 None
  • 行为: 提示用户设置环境变量或 Keyring
  • 退出码: 1
  • 示例消息:
    Error: Missing API key for provider 'deepseek' Please set DEEPSEEK_API_KEY environment variable or store it in OS Keyring

无效配置

  • 检测: TOML 解析失败
  • 行为: 显示错误行号和列号
  • 退出码: 2
  • 示例消息:
    Error: Failed to parse config.toml --> config.toml:15:10 | 15 | timeout = "invalid" | ^^^^^^^^^ expected integer

State DB 锁定

  • 检测: SQLite BUSY 错误
  • 行为: 等待 5 秒或退出
  • 退出码: 3
  • 示例消息:
    Error: State database is locked Another instance may be running Waiting 5 seconds...

Terminal 初始化失败

  • 检测: crossterm 返回错误
  • 行为: 回退到非交互模式
  • 退出码: 4
  • 示例消息:
    Error: Failed to initialize terminal Falling back to non-interactive mode Use --server flag for headless operation
