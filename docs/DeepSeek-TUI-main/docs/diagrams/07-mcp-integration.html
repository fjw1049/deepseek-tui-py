2. 工具名称编码

编码规则
格式: mcp__<sanitized_server>__<sanitized_tool>
清理规则: 字母数字+下划线 → 小写; 其他字符 → 下划线
长度处理: 如果 > 64 字符, 截断为 48 字符 + _ + 12 字符哈希
解析: parse_qualified_tool_name() 按 mcp__ 前缀和第二个 __ 分割
编码示例: 服务器: "github" 工具: "create_issue" 结果: mcp__github__create_issue 服务器: "My-Server@2024" 工具: "complex.tool.name" 结果: mcp__my_server_2024__complex_tool_name 服务器: "very-long-server-name-that-exceeds-limits" 工具: "very-long-tool-name-that-also-exceeds-limits" 结果: mcp__very_long_server_name_that_exceeds_limi_a1b2c3d4e5f6
编码算法详解
fn qualify_tool_name(server: &str, tool: &str) -> String { let sanitized_server = sanitize(server); let sanitized_tool = sanitize(tool); let qualified = format!("mcp__{}__{}", sanitized_server, sanitized_tool); if qualified.len() > 64 { let hash = compute_hash(&qualified); format!("{}_{}", &qualified[..48], &hash[..12]) } else { qualified } } fn sanitize(s: &str) -> String { s.chars() .map(|c| if c.is_alphanumeric() || c == '_' { c.to_lowercase().next().unwrap() } else { '_' }) .collect() }

3. 工具过滤机制

ToolFilter 结构体
allow: Vec<String> - 白名单模式列表
deny: Vec<String> - 黑名单模式列表
过滤逻辑
1. deny 优先级高于 allow
2. 如果 allow 为空, 所有工具通过
3. 支持 glob 风格模式 (* 通配符)
过滤示例配置: { "filter": { "allow": ["github_*", "gitlab_create_*"], "deny": ["*_delete_*", "*_destroy_*"] } } 过滤结果: ✓ github_create_issue (匹配 allow, 不匹配 deny) ✓ gitlab_create_mr (匹配 allow, 不匹配 deny) ✗ github_delete_repo (匹配 deny) ✗ gitlab_destroy_project (匹配 deny)
过滤算法实现
impl ToolFilter { fn should_include(&self, tool_name: &str) -> bool { // 1. 检查 deny 列表 for pattern in &self.deny { if glob_match(pattern, tool_name) { return false; } } // 2. 检查 allow 列表 if self.allow.is_empty() { return true; } for pattern in &self.allow { if glob_match(pattern, tool_name) { return true; } } false } }

4. JSON-RPC 2.0 协议

支持的方法

  • initialize
    与服务器进行握手, 交换能力信息
  • healthz
    健康检查, 确认服务器运行状态
  • capabilities
    查询服务器能力
  • tools/list
    列出所有可用工具
  • tools/call
    执行指定工具
  • resources/list
    列出所有可用资源
  • resources/read
    读取资源内容
  • server/list
    列出已注册的服务器
  • server/register
    注册新服务器
  • server/start
    启动服务器
  • server/stop
    停止服务器
  • server/unregister
    注销服务器
  • shutdown
    优雅关闭服务器

错误码

-32700 Parse error - JSON 解析错误
-32600 Invalid Request - 无效的请求格式
-32601 Method not found - 方法不存在
-32602 Invalid params - 参数无效
-32603 Internal error - 内部错误
JSON-RPC 请求/响应示例
请求示例 (tools/call): { "jsonrpc": "2.0", "id": 1, "method": "tools/call", "params": { "name": "create_issue", "arguments": { "title": "Bug report", "body": "Description" } } } 成功响应: { "jsonrpc": "2.0", "id": 1, "result": { "content": [ { "type": "text", "text": "Issue created: #123" } ] } } 错误响应: { "jsonrpc": "2.0", "id": 1, "error": { "code": -32602, "message": "Invalid params", "data": "Missing required field: title" } }

7. 错误处理策略

服务器启动失败
检测: 进程启动失败或初始化超时
处理: 记录错误日志, 标记状态为 Failed
恢复: 不自动重试, 等待用户手动重启
JSON-RPC 解析错误
检测: 接收到无效的 JSON 或格式错误
处理: 返回 -32700 错误码
恢复: 跳过当前消息, 继续处理后续请求
工具调用超时
检测: 工具执行超过配置的超时时间
处理: 终止工具执行, 返回超时错误给引擎
恢复: 服务器保持运行, 可继续处理其他请求
服务器崩溃
检测: 进程意外退出或无响应
处理: 标记服务器为不可用, 记录崩溃日志
恢复: 根据配置决定是否自动重启或标记为永久失败
错误处理代码示例
async fn start_server(config: &McpServerConfig) -> Result<McpClient> { // 1. 启动进程 let mut child = Command::new(&config.command) .args(&config.args) .envs(&config.env) .stdin(Stdio::piped()) .stdout(Stdio::piped()) .spawn() .map_err(|e| { error!("Failed to start server {}: {}", config.name, e); McpError::StartupFailed(e) })?; // 2. 初始化握手 (带超时) let client = timeout( Duration::from_secs(30), initialize_client(child) ).await .map_err(|_| { error!("Server {} initialization timeout", config.name); McpError::InitTimeout })??; // 3. 健康检查 client.healthz().await .map_err(|e| { error!("Server {} health check failed: {}", config.name, e); McpError::HealthCheckFailed(e) })?; Ok(client) } async fn call_tool_with_retry( client: &McpClient, name: &str, args: Value ) -> Result<ToolResult> { let mut retries = 3; loop { match timeout( Duration::from_secs(60), client.call_tool(name, args.clone()) ).await { Ok(Ok(result)) => return Ok(result), Ok(Err(e)) if e.is_retryable() && retries > 0 => { retries -= 1; warn!("Tool call failed, retrying: {}", e); sleep(Duration::from_secs(1)).await; } Ok(Err(e)) => return Err(e), Err(_) => return Err(McpError::Timeout), } } }
