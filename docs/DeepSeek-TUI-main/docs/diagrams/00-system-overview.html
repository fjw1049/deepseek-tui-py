🏗️ 子系统详细架构

1. CLI 入口

crates/cli/src/main.rs
Cli::parse() → run()
  • 使用 clap 解析命令行参数
  • 加载配置文件 (config.toml)
  • 初始化 secrets 和 state
  • 启动 TUI 或 AppServer

2. 配置系统

crates/config/src/lib.rs
Config::load()
  • 优先级: CLI参数 > 环境变量 > config.toml > 默认值
  • Profile 合并机制
  • Provider 配置 (DeepSeek/OpenAI/Anthropic)
  • 工具能力和超时配置

3. Secrets 密钥

crates/secrets/src/lib.rs
SecretStore::get()
  • 优先级: 环境变量 > OS Keyring > config.toml
  • 支持 macOS Keychain / Windows Credential Manager
  • API Key 安全存储
  • 运行时密钥注入

4. TUI 事件循环

crates/tui/src/lib.rs
EventLoop::run()
  • crossterm 事件捕获 (48ms/24ms 轮询)
  • AppMode 状态机 (Agent/Yolo/Plan)
  • 用户输入处理和命令解析
  • ratatui 渲染管线

5. Engine 引擎

crates/core/src/engine.rs
Engine::execute_op()
  • Op 调度和生命周期管理
  • 系统提示构建和 MessageRequest 组装
  • Turn Loop 和工具执行循环
  • 响应聚合和容量控制

6. LLM 客户端

crates/tui/src/client.rs
DeepSeekClient::stream()
  • HTTP POST 到 DeepSeek API
  • SSE 流式解析和 ContentBlock 跟踪
  • 重试机制 (2次透明 / 5次错误)
  • 超时控制 (90s 空闲 / 30min 总)

7. 工具系统

crates/tools/src/lib.rs
ToolRegistry::dispatch()
  • 工具注册表和能力检查
  • 能力分类: ReadOnly / WritesFiles / ExecutesCode / Network
  • 工具执行和超时控制
  • ToolResult 序列化

8. MCP 集成

crates/mcp/src/lib.rs
McpManager::discover()
  • MCP 服务器发现 (mcp.json)
  • stdio JSON-RPC 通信
  • 工具过滤 (allow/deny 列表)
  • 名称编码: mcp__server__tool

9. 审批策略

crates/execpolicy/src/lib.rs
ExecPolicyEngine::check()
  • 前缀匹配 (trusted/denied 命令)
  • 风险分级: Benign / Destructive
  • ApprovalRequest 和 UI 对话框
  • 会话级审批缓存

10. Hooks 系统

crates/hooks/src/lib.rs
HookDispatcher::emit()
  • 事件分发: Response / Tool / Job / Approval
  • Sink 类型: Stdout / Jsonl / Webhook
  • 生命周期钩子 (start/end/error)
  • 异步事件推送

11. State 持久化

crates/state/src/lib.rs
StateStore::save()
  • SQLite 会话存储
  • Checkpoint 和崩溃恢复
  • 离线队列管理
  • 历史记录查询

12. Agent 注册

crates/agent/src/lib.rs
AgentRegistry::get()
  • 模型注册表和 Provider 映射
  • 能力查询 (context/streaming/tools)
  • 模型参数配置 (temperature/top_p)
  • 多 Provider 支持

📋 关键数据结构

MessageRequest

struct MessageRequest {
    model: String,
    messages: Vec<Message>,
    tools: Vec<Tool>,
    temperature: f32,
    max_tokens: u32,
    stream: bool,
}

ContentBlock

enum ContentBlock {
    Text { text: String },
    Thinking { thinking: String },
    ToolUse {
        id: String,
        name: String,
        input: Value,
    },
}

ToolResult

struct ToolResult {
    tool_use_id: String,
    content: String,
    is_error: bool,
    metadata: HashMap<String, Value>,
}

ApprovalRequest

struct ApprovalRequest {
    command: String,
    risk_level: RiskLevel,
    context: String,
    session_id: String,
}

Op (Operation)

struct Op {
    id: String,
    prompt: String,
    mode: AppMode,
    state: OpState,
    turns: Vec<Turn>,
}

McpServer

struct McpServer {
    name: String,
    command: String,
    args: Vec<String>,
    tools: Vec<McpTool>,
}
