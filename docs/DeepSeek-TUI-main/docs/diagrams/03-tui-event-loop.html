事件类型详细说明
  • Keyboard: 键盘输入事件，包括字符输入、快捷键、方向键等
  • Mouse: 鼠标事件，包括点击、滚动、拖拽等
  • Paste: 粘贴事件，支持 bracketed paste 模式
  • Resize: 终端窗口大小变化事件
  • FocusGained / FocusLost: 终端焦点获得/失去事件

3. 用户输入处理 (User Input Handling)

ComposerState 结构

  • input: 当前输入文本内容
  • cursor_position: 光标位置（字符索引）
  • kill_buffer: Emacs 风格的剪切缓冲区（Ctrl+K / Ctrl+Y）
  • paste_burst: 粘贴突发检测，防止逐字符粘贴
  • input_history: 输入历史记录
  • history_index: 当前历史索引

输入功能

历史导航
  • Up 箭头: 向上浏览历史记录
  • Down 箭头: 向下浏览历史记录
  • 自动保存当前草稿，切换历史时不丢失
斜杠命令菜单
  • 触发: 输入 / 字符
  • 模糊匹配: 实时过滤命令列表
  • 选择: Enter 键确认选择
  • 支持的命令: /yolo, /agent, /clear, /help 等
提及菜单 (Mention Menu)
  • 触发: 输入 @ 字符
  • 内容: MCP 资源和工具列表
  • 导航: 上下箭头选择，Enter 确认
  • 自动补全文件路径和工具名称
多行输入
  • Shift+Enter: 插入换行符
  • Enter: 提交消息
  • 最大长度: 16,000 字符
  • 超出限制时自动截断并显示警告

渲染组件

  • TranscriptViewCache: 缓存已渲染的行，按单元格存储，修订号变化时失效
  • StreamingState: 跟踪流式输出状态，每个块的收集器
  • HistoryCell: 历史单元格类型（User / Assistant / System / Error / Thinking / Tool / SubAgent）
  • ApprovalDialogs: 工具批准对话框和权限提升请求
  • Overlays: 命令面板、上下文菜单、会话选择器等

鼠标/键盘事件处理

  • 鼠标滚动: 支持平滑滚动和动量滚动
  • 鼠标点击: 支持文本选择、链接点击、按钮交互
  • 文本选择: 支持鼠标拖拽选择和键盘选择
  • 复制粘贴: 支持系统剪贴板集成

BlockState 与 LineBuffer

  • 文本块: 使用换行门控，只提交完整行
  • 思考块: 绕过换行门控，实时显示
  • MarkdownStreamCollector: 缓冲并提交完整行
  • take_committable: 提取可提交的完整内容
流式渲染策略
  • 防止闪烁: 只显示完整行，避免部分单词闪烁
  • 思考模式: 实时显示推理过程，无延迟
  • 自适应分块: 根据内容类型调整提交频率
  • 缓存优化: 已提交内容缓存，减少重复渲染

性能优化策略

  • 自适应轮询: 根据活动状态动态调整轮询频率
  • 视图缓存: TranscriptViewCache 缓存已渲染内容
  • 增量渲染: 只重绘变化的区域
  • 流式优化: 换行门控减少闪烁和重绘
  • 低动作模式: 支持减少动画的无障碍模式
轮询间隔说明
  • 空闲 (48ms): 无活动时使用，约 20 FPS
  • 活跃 (24ms): 有流式输出或用户交互时，约 40 FPS
  • 动画 (80ms): 状态动画更新周期，约 12 FPS
  • 低动作模式: 空闲时延长到 120ms，减少 CPU 使用
